GESTIONE MAGAZZINO


Sorgente

La finalità del programma è quella della gestione di un magazzino nel periodo di un intero anno. Nel programma ritroviamo 20 prodotti con nome, specifica, codice identificativo, quantità e costo. Il programma, oltre a dover gestire il magazzino per 365 giorni, può ricevere in particolare 3 input e un quarto input che lo interrompe.

I 3 input sono:

1)dato un giorno, scelto dall'utente, va a visualizzare i prodotti che alla fine della giornata sono sotto scorta, ovvero meno di 5

2)dato un giorno, viene controllato se i prodotti sono stati acquistati almeno una volta

3)dato in input il codice di un prodotto, viene visualizzato quanti ne sono stati venduti in una finestra temporale di 10gg (le due date di inizio e di fine vengono scelte dall'utente)

 



 

 

Struct "Mesi" contenente i nomi dei 12 mesi dell'anno, utilizzato per visualizzare la data

Struct "Prodotti" contenente tutte le caratteristiche di un prodotto: nome, specifica, codice, quantità, prezzo

Struct "ProdottiVenduti" utilizzata dai 20 prodotti, per contenere il numero dei prodotti venduti ogni giorno

Struct "ProdottiRimanenti" utilizzata dai 20 prodotti, per contenere il numero dei prodotti rimanenti alla fine di ogni giorno

 



 

 

 

 

1) Inizializzazione function "prodotto_sotto_scorta" per visualizzare i prodotti sotto scorta di un determinato giorno

2) Inizializzazione function "prodotto_acquistato" per visualizzare i prodotti sotto scorta di un determinato giorno

3) Inizializzazione function "prodotti_venduti_10gg" per visualizzare il numero di prodotti totali venduti in 10 giorni(scelti dall'utente)

4) Inizializzazione function "menu" che visualizza il menu e l'inserimento del comando

5) Inizializzazione function "scelte" che utilizza il comando inserito nella function menu per operare una delle 4 scelte possibili



 

 

 

 

Costrutto utilizzato dal programma per vendere i prodotti e aggiungerne in caso di scarse scorte. Il blocco di istruzioni, all'inizio, mostra il giorno, il mese e la quantità del prodotto specifico disponibile all'inizio di ogni giornata, in seguito un generatore di numeri casuali, che va da 0 alla quantità di prodotto disponibile all'inizio di quella determinata giornata, genera il numero di prodotti che verranno venduti e lo inserisce nella struct "ProdottiVenduti" per ricordare la vendita in quel specifico giorno. Infine abbiamo il salvataggio dei prodotti rimanenti a fine giornata nella struct "ProdottiRimanenti", in modo tale che la quantità dei prodotti dell'inizio del giorno dopo siano equivalenti ai rimanenti del giorno prima



 

 

 

 

Function "menu" che stampa il menu e che utilizza una scanf per permette all'utente di scegliere uno dei 4 comandi. Il numero verrà associato ad un'altra variabile (command) che verrà usata dalla function scelte



 

 

Function "scelte": function che ha come scopo quello di attivare una delle altre 3 function del programma in base al numero scelto dall'utente all'avvio del programma (1 a 3, 4 per fermare il programma).

Utilizza come input:

1) command = comando scelto dall'utente e utilizzato dal programma in uno switch per utilizzare una delle 3 function

2) mesi[] = array che indica il primo giorno di ogni mese dell'anno contando i giorni da 0 a 364

3) prodottiRimanenti[] = struct dei prodotti rimanenti (per prodotto) alla fine di ogni giorno

4) ProdottiVenduti[] = struct dei prodotti venduti (per prodotto) alla fine di ogni giorno

5) prodotto[] = struct dei prodotti

Gli ultimi 4 input a loro volta sono input delle function "menu" e "scelte"



 

 

 

Function "prodotto sotto scorta": function che ha come scopo quello di, in base ad una data scelta dall'utente, visualizzare i prodotti, a fine giornata, che hanno una quantità minore di 5.

Utilizza come input:

1) mesi[] = array che indica il primo giorno di ogni mese dell'anno contando i giorni da 0 a 364

2) prodottiRimanenti[] = struct dei prodotti rimanenti (per prodotto) alla fine di ogni giorno

3) ProdottiVenduti[] = struct dei prodotti venduti (per prodotto) alla fine di ogni giorno

4) prodotto[] = struct dei prodotti

I 4 input a loro volta sono input delle function "menu" e "scelte"



 

 

 

Function "prodotto acquistato": function che ha come scopo quello di, in base ad una data scelta dall'utente, visualizzare i prodotti, a fine giornata, che non sono stati venduti.

Utilizza come input:

1) mesi[] = array che indica il primo giorno di ogni mese dell'anno contando i giorni da 0 a 364

2) prodottiRimanenti[] = struct dei prodotti rimanenti (per prodotto) alla fine di ogni giorno

3) ProdottiVenduti[] = struct dei prodotti venduti (per prodotto) alla fine di ogni giorno

4) prodotto[] = struct dei prodotti

I 4 input a loro volta sono input delle function "menu" e "scelte"



 

 

 

 

Function "prodoti venduti 10gg": function che ha come scopo quello di, in base a due date scelte dall'utente, visualizzare le vendite di un prodotto, scelto tramite codice identificativo, in 10 giorni. Utilizza come input:

1) mesi[] = array che indica il primo giorno di ogni mese dell'anno contando i giorni da 0 a 364

2) prodottiRimanenti[] = struct dei prodotti rimanenti (per prodotto) alla fine di ogni giorno

3) ProdottiVenduti[] = struct dei prodotti venduti (per prodotto) alla fine di ogni giorno

4) prodotto[] = struct dei prodotti

I 4 input a loro volta sono input delle function "menu" e "scelte"



 

 

 

Simulazione 1

Appena avviato, il programma apre il menu di scelte. Andando a selezionare la 3, si sceglie l'opzione che visualizza la quantità di uno specifico prodotto, inserendo il codice del prodotto, venduto in 10 giorni. All'inizio abbiamo la lista dei 20 prodotti con i relativi codici, si scrive uno dei codici e ci viene richiesto il giorno di inizio, il mese di inizio, il giorno della fine (massimo 10 giorni) e il mese di fine. Infine viene visualizzato il numero di prodotti venduti. Dopo ciò si può scegliere se continuare ad utilizzare il programma, andando anche ad utilizzare le altre opzioni, oppure chiuderlo.
